# 利用料金とプラン

## はじめに

Claude Codeを使い始める前に、料金体系を理解しておくことは重要です。この章では、利用可能なプラン、それぞれの特徴、そしてどのプランが自分に適しているかを判断するための情報を提供します。

## Claude Codeの料金体系の概要

Claude Codeは、大きく分けて**2つの料金方式**があります:

### 1. サブスクリプション方式（定額制）
- Claude.aiのProまたはMaxプランに加入
- **月額固定料金**で使い放題
- 初心者におすすめ
- 使用量を気にせず学習できる

### 2. API方式（従量課金制）
- Anthropic ConsoleでAPIキーを発行
- **使った分だけ課金**
- 上級者・開発者向け
- 細かいコスト管理が必要

## サブスクリプションプラン

### Claude.ai Proプラン

**料金**
- **月額$20（約2,800円）**
- 年間契約なし、月単位でキャンセル可能

**Claude Code使用制限**
- **1日あたりの制限あり**
- Opus使用: 上限に達すると自動的にSonnetに切り替わる
- Sonnet使用: より多くのリクエスト可能

**含まれる機能**
- Claude Code（ターミナル版）
- Claude.ai（Web版）へのアクセス
- Claude Sonnet 4.5の使用
- Claude Opus 4（最高性能モデル）の使用
- プロジェクト機能
- 優先アクセス（混雑時も利用可能）

**向いている人**
- Claude Codeを試してみたい初心者
- 個人開発者
- 学習目的の利用
- 月の使用量が予測しにくい人
- コスト管理をシンプルにしたい人

**実際の使用感**
```bash
# Proプランでの1日の使用例
- 午前: Opusで大規模リファクタリング（50%の上限を使用）
- 午後: 上限警告が表示される
- 自動的にSonnetに切り替わる
- Sonnetでも十分な品質で作業継続可能
```

### Claude.ai Maxプラン

**料金**
- **月額$100（約14,000円）**
- 年間契約なし、月単位でキャンセル可能

**Claude Code使用制限**
- **Proプランの5倍以上の使用量**
- Opus使用: より多くのリクエスト
- 長時間の連続作業が可能

**含まれる機能**
- Proプランの全機能
- 大幅に増加した使用量
- Claude Opus 4の長時間使用
- 優先度の高いサポート
- 新機能への早期アクセス

**向いている人**
- プロフェッショナル開発者
- 1日中Claude Codeを使う人
- 大規模プロジェクトでの使用
- チームでの共有利用
- 常にOpusモデルを使いたい人

**実際の使用感**
```bash
# Maxプランでの1日の使用例
- 終日Opusモデルで作業可能
- 大規模なリファクタリングを複数実行
- 複雑なバグ調査を長時間実行
- 上限に達することはほとんどない
```

### ProとMaxの比較表

| 項目 | Pro（$20/月） | Max（$100/月） |
|------|--------------|---------------|
| **基本使用** | ○ | ○ |
| **Opus使用量** | 制限あり | 5倍以上 |
| **Sonnet使用量** | 制限あり | 5倍以上 |
| **連続作業時間** | 数時間 | 終日可能 |
| **向いている用途** | 学習・個人開発 | プロ開発・業務 |
| **月間コスト** | 2,800円 | 14,000円 |

## API方式（従量課金）

### Anthropic Console APIキー

**料金体系**
- **使用したトークン数に応じて課金**
- モデルごとに単価が異なる

**2025年11月時点の料金（1Mトークンあたり）**

Claude Opus 4.1 (2025年8月リリース):
- 入力: $15
- 出力: $75
- ※ Opus 4.1はOpus 4のアップグレード版で、エージェントタスク、コーディング、推論に特化

Claude Sonnet 4.5 (2025年9月リリース):
- 入力: $3
- 出力: $15
- ※ 最もバランスの取れたモデル、拡張コンテキストウィンドウ(1Mトークン)対応

Claude Haiku 4.5 (2025年10月リリース):
- 入力: $1
- 出力: $5
- ※ 高速・低コストモデル

**支払い方法**
- クレジットカード登録
- 使用した分だけ月末に請求
- 無料枠なし（最初から課金）

**メリット**
- 使わなければ料金が発生しない
- 細かいコスト管理が可能
- 企業での利用に適している
- プログラムからの直接呼び出しが可能

**デメリット**
- 使用量の予測が難しい
- トークン数の計算が複雑
- 月末にならないと正確な金額がわからない
- 初心者には管理が難しい

**コスト例**

```bash
# 想定: 1日4時間、Claude Codeを使用

1日の使用量概算:
- 入力トークン: 約100,000トークン
- 出力トークン: 約50,000トークン

Opus使用時（1ヶ月30日）:
- 入力: 100K × 30 × $15/1M = $45
- 出力: 50K × 30 × $75/1M = $112.5
- 合計: $157.5（約22,000円）

Sonnet使用時（1ヶ月30日）:
- 入力: 100K × 30 × $3/1M = $9
- 出力: 50K × 30 × $15/1M = $22.5
- 合計: $31.5（約4,400円）
```

**向いている人**
- 使用量が少ない人
- 月に数回しか使わない人
- 企業での導入
- APIを直接制御したい上級者
- 正確なコスト管理が必要な人

## モデルの選択と使い分け

### Claude Opus 4（最高性能モデル）

**特徴**
- 最も高性能なモデル
- 複雑な推論が得意
- コード品質が高い
- 料金が高い

**向いているタスク**
- 大規模リファクタリング
- アーキテクチャ設計
- 複雑なバグ調査
- 高品質なコード生成

**使用例**
```bash
> /model
# 「Opus」を選択

> プロジェクト全体をTypeScriptに移行してください
```

### Claude Sonnet 4.5（バランス型モデル）

**特徴**
- コストと性能のバランスが良い
- 応答速度が速い
- 通常のタスクには十分な品質
- 料金が安い（Opusの1/5）

**向いているタスク**
- 日常的なコーディング
- 簡単なバグ修正
- ドキュメント作成
- コードレビュー

**使用例**
```bash
> /model
# 「Sonnet」を選択

> README.mdを作成してください
```

### モデル選択の実践的な使い分け

```bash
# デフォルト設定（推奨）
> /model
# 「Default」を選択
# → 上限の50%まではOpus、それ以降はSonnetに自動切り替え

# 品質重視の場合
> /model
# 「Opus」を選択
# → 常に最高品質（上限に注意）

# コスト重視の場合
> /model
# 「Sonnet」を選択
# → 日常作業には十分
```

## どのプランを選ぶべきか

### 選択フローチャート

```
あなたの状況は？
│
├─ 初めてClaude Codeを使う
│  └→ 【Proプラン】がおすすめ
│     理由: 定額で安心、学習に集中できる
│
├─ 個人開発者・週に数回使う
│  └→ 【Proプラン】がおすすめ
│     理由: コストパフォーマンスが良い
│
├─ プロ開発者・毎日使う
│  └→ 【Maxプラン】がおすすめ
│     理由: 制限を気にせず作業できる
│
├─ 企業での導入
│  └→ 【APIキー方式】がおすすめ
│     理由: コスト管理、監査が可能
│
└─ 月に数回しか使わない
   └→ 【APIキー方式】も検討
      理由: 使わない月は料金が発生しない
```

### 具体的な使用パターン別推奨

**学生・初心者**
- 推奨: **Proプラン（$20/月）**
- 理由: 定額で安心、学習に最適

**フリーランス開発者**
- 推奨: **Proプラン** または **Maxプラン**
- 判断基準: 1日の使用時間
  - 2-3時間: Pro
  - 4時間以上: Max

**スタートアップ**
- 推奨: **Maxプラン** または **APIキー**
- 判断基準: チーム規模
  - 1-2人: Max（共有可能）
  - 3人以上: APIキー（各自管理）

**大企業**
- 推奨: **APIキー方式**
- 理由: コスト管理、セキュリティ、監査が必要

## 料金を節約するTips

### 1. モデルの使い分け

```bash
# 日常作業はSonnet
> /model → Sonnet選択

# 重要な作業だけOpus
> /model → Opus選択
> 複雑なリファクタリング実行
> /model → Sonnet選択に戻す
```

### 2. /compactで トークン節約

```bash
# 会話が長くなったら
> /compact
# これまでの会話を要約して、トークン数を削減
```

### 3. 適切なタイミングで/clear

```bash
# タスクが変わったら
> /clear
# 不要なコンテキストをリセット
```

### 4. 明確な指示で試行錯誤を減らす

```bash
# 悪い例（何度もやり直しが必要）
> ボタンを作って

# 良い例（一度で完成）
> 青色の送信ボタンを作ってください。
> サイズは40px高さで、Tailwind CSSを使用。
> クリック時に disabled になり、ローディングスピナーを表示。
```

## 無料で試す方法

Claude Code自体に無料プランはありませんが、以下の方法で試すことができます:

### 1. Claude.ai無料プラン + API

- Claude.aiは無料プランあり（制限付き）
- APIキーを使ったClaude Code利用は可能
- 少額の従量課金で試せる

### 2. 初月のみProプラン

- $20で1ヶ月試す
- 合わなければすぐキャンセル可能
- 返金はないが、リスクは$20のみ

## 支払い方法と注意点

### 対応している支払い方法

- クレジットカード（Visa、Mastercard、American Express）
- デビットカード
- PayPal（一部の地域）

### 請求タイミング

**サブスクリプション（Pro/Max）**
- 月の初日に自動課金
- キャンセル: 次回更新日まで利用可能

**API方式**
- 月末締め
- 翌月初旬に請求

### 注意点

**為替レート**
- ドル建てのため、為替の影響を受ける
- 円安時は実質値上がり

**自動更新**
- サブスクリプションは自動更新
- 不要になったら早めにキャンセル

**使用量の監視**
- APIキー方式は使いすぎに注意
- コンソールで定期的に確認

## まとめ

Claude Codeの料金プラン選択のポイント:

**初心者・個人開発**
- → **Proプラン（$20/月）**
- 定額で安心、学習に最適

**プロフェッショナル**
- → **Maxプラン（$100/月）**
- 終日使える、制限なし

**企業・上級者**
- → **APIキー（従量課金）**
- 細かい管理が可能

**モデル選択**
- 重要な作業: Opus
- 日常作業: Sonnet
- 自動選択: Default

まずはProプランで始めて、使用状況に応じてMaxやAPIキーに移行するのがおすすめです。
